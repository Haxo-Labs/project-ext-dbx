# DBX Documentation

Welcome to the DBX documentation! DBX is a lightweight, high-performance database API server with backend-agnostic architecture.

## What is DBX?

DBX is a minimal and portable HTTP/WebSocket proxy that exposes multiple database types through a unified API layer. Built in Rust with pluggable backend adapters implementing the `UniversalBackend` trait, DBX provides consistent APIs regardless of the underlying database.

### Key Features

- **Backend-Agnostic**: Pluggable adapter system for multiple database types
- **Unified Interface**: Consistent HTTP REST API and WebSocket support across all backends
- **TypeScript SDK**: Client library with backend capability detection
- **High Performance**: Built in Rust for maximum efficiency
- **Lightweight**: Minimal footprint, perfect for edge computing
- **Extensible**: Easy to add new database backends

### Supported Backends

#### Currently Available
- **Redis**: Complete implementation with strings, hashes, sets, sorted sets, bitmaps

#### Planned Backends
- **MongoDB**: Document-based operations with collections and aggregations
- **PostgreSQL**: Relational database with SQL query support
- **SQLite**: Embedded database for local storage

## Quick Start

### Using Docker

```bash
# Run with Redis backend
docker run -d --name dbx -p 3000:3000 \
  -e BACKEND_TYPE=redis \
  -e REDIS_URL=redis://localhost:6379 \
  effortlesslabs/dbx:latest

# Run with auto-detection
docker run -d --name dbx -p 3000:3000 \
  effortlesslabs/dbx:latest
```

### Using TypeScript SDK

```bash
# Install SDK
npm install dbx

# Use in your code
import { DbxClient } from "dbx";

const client = new DbxClient("http://localhost:3000");

// Data operations
await client.data.set("user:1", { name: "Alice", age: 30 });
const user = await client.data.get("user:1");

// Backend-specific operations (when available)
if (client.capabilities.supports("redis_strings")) {
  await client.redis.string.set("key", "value", 3600);
}
```

## API Endpoints Overview

### Backend-Agnostic Endpoints

| Endpoint              | Method | Description                    |
| --------------------- | ------ | ------------------------------ |
| `/data/{key}`         | GET    | Get data by key                |
| `/data/{key}`         | POST   | Set data by key                |
| `/data/{key}`         | DELETE | Delete data by key             |
| `/query`              | POST   | Execute query operation        |
| `/stream`             | POST   | Stream operation               |
| `/admin/health`       | GET    | Health check                   |
| `/admin/capabilities` | GET    | Backend capabilities discovery |

### Backend-Specific Endpoints

#### Redis (when Redis backend is active)

| Endpoint                          | Method | Description      |
| --------------------------------- | ------ | ---------------- |
| `/redis/string/{key}`             | GET    | Get string value |
| `/redis/string/{key}`             | POST   | Set string value |
| `/redis/hash/{key}/field/{field}` | GET    | Get hash field   |
| `/redis/set/{key}/members`        | GET    | Get set members  |

### WebSocket Endpoints

| Endpoint                     | Description       |
| ---------------------------- | ----------------- |
| `ws://localhost:3000/data`   | Data operations   |
| `ws://localhost:3000/query`  | Query operations  |
| `ws://localhost:3000/stream` | Stream operations |

## Architecture

### Core Components

- **dbx-core**: Backend traits and operation abstractions
- **dbx-adapter**: Database-specific implementations of UniversalBackend
- **dbx-config**: Multi-backend configuration management
- **dbx-router**: HTTP/WebSocket routing with backend abstraction

### UniversalBackend Trait

All database adapters implement a consistent interface:

```rust
#[async_trait]
pub trait UniversalBackend: Send + Sync {
    async fn execute_data(&self, operation: DataOperation) -> Result<DataResult, DbxError>;
    async fn execute_query(&self, operation: QueryOperation) -> Result<QueryResult, DbxError>;
    async fn execute_stream(&self, operation: StreamOperation) -> Result<StreamResult, DbxError>;
    fn capabilities(&self) -> BackendCapabilities;
}
```
